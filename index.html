Automated weekly refresh — architecture

This dashboard is built to run fully automated using free or low-cost tools. The diagram below shows the complete data pipeline from BLS/Lightcast through to the live webpage your students see every Monday.

Sample automation script

This Python script runs on a GitHub Actions cron job every Monday at 6 AM. It fetches BLS OEWS data, transforms it, and pushes updated data.json to your repo — triggering an automatic page rebuild.

# weekly_refresh.py — runs via GitHub Actions every Monday import requests, json from datetime import datetime BLS_API_KEY = "YOUR_BLS_API_KEY" # free at bls.gov/developers SERIES_IDS = [ "OEUS211713000000015112", # Baltimore Metro occupation wages "OEUS211713000000013111", # Management analysts ] def fetch_bls(series): url = "https://api.bls.gov/publicAPI/v2/timeseries/data/" payload = {"seriesid": series, "registrationkey": BLS_API_KEY, "startyear": "2023", "endyear": "2026"} r = requests.post(url, json=payload) return r.json()["Results"]["series"] data = fetch_bls(SERIES_IDS) output = {"refreshed": datetime.utcnow().isoformat(), "series": data} with open("data/labor_market.json", "w") as f: json.dump(output, f, indent=2) print(f"✓ Refreshed {len(data)} series at {output['refreshed']}")
